Getting Started

1. Obtain an OpenAI API Key

  1. Visit the OpenAI website at platform.openai.com/playground.
  2. Go to your dashboard.
  3. Navigate to the API Keys section.
  4. Create a new secret key and give it a descriptive name.
  5. Copy the generated key and store it securely in an offline location like a text file.
  6. Important: You will not be able to see the key again after clicking the "Done" button. If you lose the key, you'll need to generate a new one.

2. Setup Project Files

  1. Create three empty files in your project directory:
    • index.html
    • script.js
    • styles.css
  2. Rename script.js to chatbot.js.

3. Generate Code with ChatGPT

  1. Go to ChatGPT and ask it to write a simple chatbot with an HTML interface using OpenAI's API and JavaScript.
  2. Request that the HTML and JavaScript be separated into separate files.
  3. Copy the HTML code into index.html.
  4. Copy the CSS code into styles.css.
  5. Copy the JavaScript code into chatbot.js.

4. Insert Your API Key

  1. In the chatbot.js file, find the line where it says "your API key".
  2. Replace this placeholder with the API key you generated earlier.

5. Run the Chatbot

  1. Open index.html in Visual Studio Code.
  2. Right-click on the index.html file and select "Go Live" to open the HTML file in your web browser.
  3. You should see your chatbot interface!

How The Chatbot Works

User Input

The user types a message in the provided text area of the chatbot interface.

Sending the Message

When the user clicks "Send", the JavaScript code sends the message to the OpenAI API.

  • The code formats the message as a JSON file.
  • The code also includes the user's API key, so OpenAI can charge for using the model.

Generating a Response

OpenAI's API receives the user's message and uses its language model to create a response.

Returning the Response

The API sends its response back to the JavaScript code.

Displaying the Response

The JavaScript code displays the API's response on the HTML page.

Conversation Memory

In its initial version, this chatbot doesn't have any memory of the conversation, meaning each question and answer is treated as a fresh start. This can be modified.

Security Considerations

Protect Your API Key

Your API key should be treated like a password.

  • Don't share your API key publicly.
  • Don't store it in code that is publicly accessible.
  • If using a school computer, do not store your API key in your code on that computer.

Set Usage Limits

To prevent unexpected charges, set usage limits and disable automatic refills in your OpenAI account settings. This way, even if your key is compromised, the financial impact will be limited.

Hiding Your API Key

Techniques exist to hide your API key when deploying your chatbot to the web. These techniques ensure others can interact with your chatbot without having access to your key. You will learn more about this later.

Additional Key Elements, Process Notes, and Tradeoffs

System Instructions

Purpose: System instructions function like a set of guidelines or a mission statement for the language model. They are provided alongside your prompt and operate behind the scenes, influencing how the model generates its completion.
  • Impact: Carefully crafted system instructions can significantly improve the chatbot's output, enabling it to:
    • Adopt a specific persona (e.g., a British tea snob)
    • Follow particular formatting rules (e.g., provide code in JavaScript syntax)
    • Achieve desired outcomes (e.g., generate website HTML for tea brands)
  • Experimentation: The effect of system instructions can be unpredictable. Sometimes, a change in the system instructions might have a dramatic impact, while other times, the difference might be negligible. Experimentation and comparison are crucial for finding effective system instructions.
  • Implementation: In the JavaScript method demonstrated, you would integrate your system instructions into the message sent to the model.

Temperature

Definition: The temperature parameter controls the randomness of the language model's output.
  • Range: The temperature typically ranges from 0 to 2.
  • Impact:
    • Higher temperature: (closer to 2) Results in more creative and unpredictable responses, potentially introducing unexpected languages, errors, or unusual formats.
    • Lower temperature: (closer to 0) Leads to more predictable and deterministic responses, often sticking to expected patterns and formats. A temperature of 0 aims for maximum predictability, but even then, the same prompt might not always produce the identical output.
  • Default Value: The default temperature is usually set to 1.

Prompting Strategies

  • Moving Beyond Questions: Traditional search engines have conditioned us to interact using questions. However, when working with language models, using direct commands can be more effective.
  • Example: Instead of "Can you give me a recipe?", try a more authoritative approach, such as "You must give me a recipe," to guide the model toward the desired outcome.

Tradeoffs: OpenAI API vs. Hugging Face Spaces

OpenAI API

  • Advantages:
    • Access to powerful models like GPT-3.5 and GPT-4.
    • Greater control over customization.
  • Disadvantages:
    • Requires a paid subscription.
    • More complex setup and potential security concerns regarding API key management.

Hugging Face Spaces

  • Advantages:
    • Free to use.
    • Easy setup through a user-friendly interface.
    • Automatic hosting on the internet.
  • Disadvantages:
    • Relies on Python, which is not covered in the initial part of the course.
    • May have usage limitations.
    • Potentially less capable models compared to OpenAI's offerings.

Process Note: Incorporating Memory

  • Stateless by Default: The initial chatbot example is "stateless," meaning it does not retain any memory of previous interactions. Each prompt is treated in isolation.
  • Adding Memory: To provide the chatbot with memory, you need to modify the code to:
    • Keep track of the entire conversation history.
    • Include this conversation history as part of every prompt sent to the model.
  • Challenges:
    • As the conversation lengthens, sending the entire history can become computationally expensive and potentially exceed token limits.
    • Sophisticated memory management techniques may be required for extended conversations.

Chatbot Project Overview: Key Information and Specifics

Objective

  • Your team will build a chatbot using JavaScript and a chatbot framework, with the freedom to choose among options like OpenAI API, Hugging Face Spaces, or other suitable frameworks.
  • The minimum requirement is for the chatbot to embody either:
    • Personality: This could involve adopting a distinct voice, tone, and style in its responses.
    • Area of Knowledge: The chatbot could be specialized in a particular domain, such as cooking, history, or technology, providing knowledgeable answers within that area.
    • Perspective: The chatbot might present information from a specific viewpoint or ideology, influencing how it interprets and responds to user input.

Collaboration and Teamwork

Key Considerations

  • Understanding System Instructions: System instructions are crucial for shaping the chatbot's behavior and responses. Experimentation and comparison are needed to find effective instructions.
